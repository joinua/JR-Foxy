# Реліз і оголошення оновлення

Оновлення JR-Foxy запускаються лише через workflow **Release and deploy** у GitHub Actions. Він:

1. оновлює сервер до точного commit SHA запуску;
2. перебудовує Docker-образ і очікує Docker healthcheck;
3. створює GitHub Release з номером та описом змін;
4. надсилає повідомлення в чат Офіцерів.

Повідомлення надсилається тільки після статусу `healthy`. Простий restart контейнера, аварійний перезапуск або перезавантаження сервера повідомлення не створюють.

## Одноразове налаштування

У GitHub: **Settings -> Secrets and variables -> Actions -> New repository secret** додайте:

| Secret | Значення |
|---|---|
| `DEPLOY_HOST` | IP-адреса або домен сервера |
| `DEPLOY_USER` | SSH-користувач для деплою |
| `DEPLOY_PATH` | Абсолютний шлях до клонованого репозиторію на сервері, наприклад `/opt/JR-Foxy` |
| `DEPLOY_SSH_KEY` | Приватний ключ deploy-ключа у форматі OpenSSH |
| `DEPLOY_KNOWN_HOSTS` | Вивід `ssh-keyscan -H ваш_сервер`, отриманий і перевірений з довіреного комп'ютера |
| `TELEGRAM_BOT_TOKEN` | Токен JR-Foxy від BotFather |
| `OFFICERS_CHAT_ID` | ID чату Офіцерів |
| `OFFICERS_THREAD_ID` | ID гілки Офіцерів; якщо повідомлення потрібне в загальний чат, secret не створюйте |

На сервері:

1. Клон репозиторію має належати `DEPLOY_USER`, а його remote `origin` повинен мати доступ на читання до GitHub.
2. У каталозі `DEPLOY_PATH` має бути робочий `.env` з поточними секретами бота.
3. Має працювати `docker compose`.
4. Публічний ключ із пари для `DEPLOY_SSH_KEY` треба додати до `~/.ssh/authorized_keys` користувача `DEPLOY_USER`.

## Як випустити версію

1. Змерджте потрібні зміни до `main`.
2. Відкрийте **Actions -> Release and deploy -> Run workflow**.
3. Вкажіть версію, наприклад `v1.0.0`.
4. У полі `changes` напишіть короткий список українською, наприклад:

   ```
   • Виправлено підрахунок активності в Родині JR
   • Додано логування для діагностики
   ```

Не запускайте звичайний ручний деплой для змін, які мають бути оголошені: тоді не буде ні release, ні повідомлення.

## Healthcheck

Контейнер стає `healthy` після успішного `getMe` до Telegram і регулярного heartbeat основного event loop. Якщо процес зависне, файл heartbeat за 90 секунд застаріє, а Docker позначить контейнер як `unhealthy`.
